## 📝 번역 가이드라인

### 1. 번역 원칙

- **일관성**: 기존 번역된 템플릿 파일의 용어 및 스타일 유지
- **정확성**: 기술 용어의 정확한 의미 전달
- **자연스러움**: 한국어로 자연스럽게 읽히도록 표현
- **완전성**: 모든 설명, 예시, 주석 포함

### 2. 용어 통일

| 영문                | 한국어                   | 비고 |
| ------------------- | ------------------------ | ---- |
| Agent               | 에이전트                 |      |
| Task                | 작업                     |      |
| Template            | 템플릿                   |      |
| Workflow            | 워크플로우               |      |
| Checklist           | 체크리스트               |      |
| Story               | 스토리                   |      |
| Epic                | 에픽                     |      |
| Sprint              | 스프린트                 |      |
| PRD                 | PRD (제품 요구사항 문서) |      |
| Architecture        | 아키텍처                 |      |
| Acceptance Criteria | 승인 기준                |      |
| Elicitation         | 유도                     |      |

### 3. 번역 보존 항목

다음 항목은 번역하지 않고 원문 유지:

- 파일명 (`.md`, `.yaml`)
- YAML 키 이름 (`name:`, `id:`, `commands:`)
- 코드 블록 내용
- URL 및 경로
- 변수명 및 플레이스홀더 (`{{variable}}`)

### 4. 번역 형식

- **YAML 블록**: 값(value)만 번역, 키(key)는 유지
- **마크다운 제목**: 완전 번역
- **코드 예시**: 주석만 번역, 코드는 유지
- **LLM 지침**: `[[LLM: ...]]` 블록 내용 완전 번역
